千年ビルダー

千年ビルダーは、電子カルテなどの医療情報システムに簡単なAPIを提供し、千年カルテが必要とするMML4.0形式のデータを生成するAPIサーバーです。

利用イメージ

  • 患者さんがクリニックを受診しました。
  • 医師は処方せんを出しました。

クライアントの医療情報システムはこの時の関係（患者、医師、処方）を簡単なJSONで表し、サーバーへポストします。サーバーはMML4.0にエンコードされたデータを返却します。下記は、左がAPI、右が生成されたMMLです。

API:

サーバーが生成した MML4.0

　

サンプルデータの個人名にはなんちゃって個人情報を使用しています。

APIを使用するメリット

  • MMLを直接サポートする場合に比べ、開発の時間が大幅に短縮されます。
  • 千年カルテに固有なデータ設定が隠蔽されているため、医療情報自体に集中できます。
  • MML/openEHR の理解に役立ちます。

千年ビルダーの特長

  • 医療情報システムの種類（業務の種類やプログラムの実装言語）に依存しません。
  • 病院の規模を問わず、大病院からクリニックまでカバーします。
  • 安全に運用できます。

API資料

  • APIのスタイルはREST、データ形式はJSONです。
  • プロジェクトに参加するとこれと同等のサーバーが SaaS で提供されます。
  • Wiki に仕様及びガイドがあります。
  • GitHub にクライント及びサーバーのソースコードが置いてあります。
  • MML インスタンスのバリデーション・チェックはこちらで行うことができます。
